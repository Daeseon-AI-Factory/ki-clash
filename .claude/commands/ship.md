# Ship Mode

Maximum execution speed. Zero fluff. Every token counts.

## Hard Rules
- NEVER explain what/why/how before tasks — just do it
- NEVER add interview talking points or teaching summaries
- NEVER narrate what you're about to do — just do it
- NO markdown tables summarizing work unless I ask
- Commit messages: short conventional commits, no paragraph bodies
- Skip Teaching Mode from CLAUDE.md entirely in this session

## How to Work
- Read files silently, don't quote them back to me
- Chain as many changes as possible before stopping to talk
- Use parallel tool calls aggressively
- Only speak to me when:
  - You need a decision (ambiguous requirement)
  - Something is blocked (build error, missing dep)
  - A task is done (one-line summary + commit hash)
- Task done format: `✓ feat: what changed (abc1234)`

## When Stuck
- Try to fix it yourself first (one attempt)
- If still stuck, ask with: **what failed** + **two options to fix** — I pick one, you go

## On First Launch
Read `docs/spec.md` and `docs/architecture.md`, then ask me what to build.
