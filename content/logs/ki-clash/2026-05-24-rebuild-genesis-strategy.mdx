## Decisions

**DR-1 — Backend language for the game server (Python vs Spring vs Go).**
Chosen: Option D, a hybrid 2-tier split — Python/FastAPI platform server now (auth, payment, profile, history), Go game server later once DAU exceeds ~500. Why: Spring is a known game-server anti-pattern and breaks Factory ROI (LangChain4j lags Python LangChain by 6-12 months); Python is good-enough CRUD for the platform tier; Go's goroutine model (~10KB/conn, <1ms GC, 50-100k conns) is only needed at scale, so the rewrite is deferred under "Build for 100, optimize later" (log lines 200-208).

**DR-2 — Visual aesthetic (pixel art vs modern card game).**
Chosen: Option B, a modern anime card-game aesthetic with AI-generated assets (Firefly + Figma), all original art, high-impact arcade effects. Why: Ki Clash monetizes via ads, so first-5-second visual impact drives the funnel; the initial pixel-art system in `web/src/components/pixel-art/` reads "indie cheap" on mobile, outsourced art ($300-800/character) breaks the Factory, and geometric is too far from the ki-battle target — anime + AI gives a high ceiling, low marginal cost, and compounding skill (log lines 210-214).

## Numbers behind DR-1 (from the log)

- WebSocket memory: ~100KB/conn Python & Spring vs ~10KB Go (2KB goroutine stack)
- GC pause: Python 0, Spring 1-10ms, Go <1ms
- Concurrent connections: Python/Spring 5-10k vs Go 50-100k practical
- Startup time: Python 200ms, Spring 3-10s, Go <50ms
- Container image: Python 200MB+, Spring 300MB+, Go 30-50MB
- Net: Go gives 5-10x WebSocket connection density
- Counterweight: FastAPI handles 5k req/s on one instance, and Python ships CRUD features 2-3x faster than Go — hence Python stays on the platform tier
- Cost framing: outsourced art $300-800/character → $3-8k across 10 products
